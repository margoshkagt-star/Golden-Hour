# COMMANDS — референс всех команд бота

> Полный список команд Telegram-бота с описанием, аргументами, правами и примерами вывода.

## Быстрый референс

| Команда | Аргументы | Права | Что делает |
|---|---|---|---|
| `/start` | — | owner | Регистрация владельца (chat_id) + приветствие + список команд |
| `/help` | — | все | Показывает команды (та же логика что /start) |
| `/today` | — | owner | Заметки за сегодня |
| `/week` | — | owner | Заметки за последние 7 дней |
| `/digest` | — | owner | Непрочитанные с последней рассылки; обновляет `digest.last_digest_at` |
| `/search` | `<текст>` | owner | Поиск substring по content |
| `/info` | `[scope]` или `[mine]` или `[stats]` | team+ | Категоризированная витрина / личная сводка / статистика |
| `/ideas` | `[scope]` | team+ | Чистый список валидных идей (без мусора) |
| `/skills` | `[scope]` | owner | Сгенерировать драфты SKILL.md локально (fallback) |
| `/rejected` | `[category]` | team+ | Отсеянные идеи (сводка или по категории) |
| `/split` | `<текст идеи>` | team+ | Разбить комплексную идею на под-идеи |
| `/classify` | `[new-only]` | team+ | Прогнать intake, обновить `memory/ideas.md` |

**scope:** `all` (по умолчанию) | `today` | `week` | `month`

**category (для `/rejected`):** `command` | `test_input` | `media_no_caption` | `empty` | `duplicate` | `manual` | `unknown`

## Команды владельца (owner-only)

### `/start`

Регистрация владельца. При первом вызове бот записывает `chat_id` и `user_id` в `bot-config.json → owner`. При последующих — повторяет приветствие.

```
/start
→ Привет, хозяин! 👋
  Твой chat_id = 1038917447 записан.

  Команды:
  /today — заметки за сегодня
  ...
```

Если вызвал НЕ owner (по username):
```
/start
→ Принято ✅
  Я — бот для сбора заметок. Просто пришли текст или голосовое, и я сохраню.
```

---

### `/today`

Все заметки за сегодня (по Москве).
```
/today
→ 📒 Заметки за сегодня

  • 14:30 — @beatusx (id 1038917447) 💬 text: «Идея — присылать мемы по настроению»
  • 14:35 — @svirepymedved (id 5649925712) 💬 text: «Добавить котиков»
  • 14:42 — @beatusx 💬 text: «Сколько стоит VPS сервер?»

  Итого: 3 заметок
```

---

### `/week`

То же, за последние 7 дней.

---

### `/digest`

Все непрочитанные заметки с момента `digest.last_digest_at` (в `bot-config.json`). После отправки обновляет timestamp. Если ничего нового — короткий ответ "(пусто)".

Полезно как «утренняя рассылка» — вызывается автоматически каждые `digest.interval_hours` (по умолчанию 3).

---

### `/search <текст>`

Substring-поиск по `content` во всём `notes.jsonl`. Case-insensitive.

```
/search мем
→ 🔍 Поиск: «мем»

  • 2026-06-23 14:30 — @beatusx 💬 text: «Идея — присылать мемы по настроению»
  • 2026-06-22 11:15 — @beatusx 💬 text: «мемы про учёбу»

  Итого: 2 заметок
```

---

### `/skills [scope]`

Запустить **локальный forge-fallback**: `idea_to_skill.run_pipeline(scope)`. Создаёт драфты SKILL.md в `workspace/Golden-Hour/skills/<slug>/` (если папка существует).

**Без forge-skill агента** — это единственный способ сгенерировать SKILL.md (без research/тестов).

**С forge-skill агентом** — обычно используется авто-пайплайн через `forge_queue.jsonl` (не нужно вызывать вручную).

```
/skills week
→ 🛠 Сгенерировано 5 драфтов SKILL.md:

  • meme-by-mood → workspace/Golden-Hour/skills/meme-by-mood/SKILL.md
  • vps-cost-info → workspace/Golden-Hour/skills/vps-cost-info/SKILL.md
  • task-decomposition → workspace/Golden-Hour/skills/task-decomposition/SKILL.md
  • notification-grouping → workspace/Golden-Hour/skills/notification-grouping/SKILL.md
  • voice-to-idea → workspace/Golden-Hour/skills/voice-to-idea/SKILL.md
```

## Команды владельца и команды (team+)

### `/info [scope|mine|stats]`

**Без аргументов** — категоризированная витрина ВСЕХ хороших идей команды, отсортированных по категориям (Wellness / Образование / Продуктивность / ...). Мусор и тестовые записи отфильтрованы. Реализованные идеи (есть SKILL.md в staging/prod) помечаются ✅.

```
/info
→ 📚 Витрина идей команды (все)
  📊 27 хороших идей в 8 категориях · 2026-06-16 → 2026-06-23
  _Отфильтровано 13 шумных / тестовых записей_

  ─── 🤖 Автоматизация и скиллы (5) ───

  • «Бот должен уметь подсчитывать повторяющиеся задачи»
    2026-06-17 01:27 · @beatusx · ⭐2

  ─── ⚡ Продуктивность и задачи (4) ───
  ...

  ✅ Уже реализовано скиллами: 4 из 27
```

**Аргумент `mine`** — только твои идеи, группировка по дате, с важностью, темой и полным текстом (с мягкой обрезкой если очень длинно).

```
/info mine
→ 📊 Твои идеи (все) — @beatusx (id 1038917447)
  📈 14 шт. · 2026-06-16 → 2026-06-23 · средняя важность 2.4 ⭐
  🏷 Топ-темы: автоматизация задач (3), здоровье (2), обучение (2)

  ─── 2026-06-23 (3 шт.) ───

  💡 14:30 · важность 3/5 ⭐⭐⭐ · 💬
     🏷 автоматизация задач
     «Бот должен уметь подсчитывать повторяющиеся задачи»
```

**Аргумент `stats`** (или `team`, `all`, `everyone`) — статистика по всей команде: топ-авторов, по дням, по типам.

**Аргумент `today|week|month`** — фильтр по дате.

---

### `/ideas [scope]`

Список ВСЕХ валидных идей (без мусора, без тестовых записей). Дедуп по `(user_id, content)` — оставляет только новейшую запись из каждой группы.

```
/ideas week
→ 💡 Идеи (за неделю) — 8 шт.

  • 2026-06-23 14:30 — @beatusx 💬: «Идея — присылать мемы по настроению»
  • 2026-06-22 16:40 — @beatusx 💬: «супер, теперь научись воспринимать транскрибированные сообщения как идеи»
  • 2026-06-21 16:04 — @beatusx 💬: «Идея боту напоминать про здоровые активности, например после активной сессии решения задач по физике или математике предложить потрогать траву»
      └─ 1. Бот напоминает про здоровые активности
      └─ 2. После активной сессии решения задач
      └─ 3. Предложить потрогать траву
  ...
```

Записи, разбитые на под-идеи через `/split`, показываются с под-нумерацией.

---

### `/rejected [category]`

Просмотр отсеянных идей. **Без аргумента** — сводка по категориям + топ-авторов. **С аргументом** — список записей в конкретной категории.

```
/rejected
→ 🗑 Отсеянные идеи (13 шт.)

  По категориям:
  • command: 7 (54%)           ← /info, /start, etc.
  • test_input: 3 (23%)        ← ппп, тест, аааа
  • manual: 2 (15%)            ← помечены is_idea=false
  • media_no_caption: 1 (8%)   ← фото без подписи

  Топ-авторов отсева:
  • @beatusx — 9
  • @svirepymedved — 3
  • @aabarabanov — 1

  Использование: /rejected <category>
  Доступные: command, test_input, media_no_caption, empty, duplicate, manual, unknown
```

```
/rejected command
→ 🗑 command (7 шт.)

  • 2026-06-17 10:22 — @beatusx 💬: «/info»
  • 2026-06-17 10:22 — @beatusx 💬: «/ideas»
  ...
```

### `/split <текст идеи>`

Разбить комплексную идею на атомарные под-идеи. Сплиттер детектит:
- Маркированные списки (`*`, `-`, `•`)
- Разделители по `и`, `также`, `а ещё`, `+`
- Разделители по `.!?` + заглавная буква

```
/split Бот должен понимать настроение, присылать мемы и помогать с CI
→ 🪓 Разбивка идеи — 3 под-идеи:

  Исходная идея:
    «Бот должен понимать настроение, присылать мемы и помогать с CI»

  Под-идеи:
    1. Бот должен понимать настроение
    2. Присылать мемы
    3. Помогать с CI
```

Если идея атомарная:
```
/split Просто текст без разделителей
→ 🪓 Разбивка идеи

  Исходная идея:
    «Просто текст без разделителей»

  _Идея выглядит атомарной — сплиттер не нашёл под-идей._
  Попробуй переформулировать:
    • через «и», «также», «а ещё»;
    • разбить на 2-3 предложения (по точкам).
```

**Важно:** `/split` только ПОКАЗЫВАЕТ разбивку. Чтобы сохранить под-идеи — отправь их текстом с темой.

---

### `/classify [new-only]`

Прогнать intake-классификатор по `notes.jsonl` и обновить `memory/ideas.md` + `memory/ideas_rejected.md`.

**Без аргумента** — полная переклассификация (все записи с начала).

**С аргументом `new-only`** — только записи после `ideas_state.json → last_run_ts`.

```
/classify new-only
→ 🧮 Классификация (только новые)

  📥 Записей: 5
  💡 Идей: 3
  🗑  Отсеяно: 2

  По категориям:
    • Продуктивность и задачи: 2
    • Автоматизация и скиллы: 1

  ✅ Идеи — memory/ideas.md
  ✅ Отсеянные — memory/ideas_rejected.md
```

## Гостевой режим (без команд)

Любой пользователь Telegram, который не owner и не в team, может:
- Прислать **текст** → `append_note(kind="text", content=...)` → ответ "Принято ✅"
- Прислать **голосовое** → если Whisper включён, транскрибируется и сохраняется как text. Иначе — placeholder. Ответ: "Принято ✅" (с транскриптом) или "Записано 🎙 (Xs) — без транскрипта" (без).
- Прислать **фото** → `append_note(kind="photo", content=caption or "(фото без подписи)")` → ответ "Принято ✅"
- Прислать **документ** → `append_note(kind="document", content="<file_name> — <caption?>")` → ответ "Принято ✅"

Гостевые сообщения **записываются в `notes.jsonl`** и видны в `/info` как обычные идеи (если проходят intake-фильтр). Это by design — не теряем сигнал.

## Права (RBAC)

```python
# telegram_notes_bot.py
def is_owner(user):       # username match
def is_team_member(user): # owner OR username in team
```

Хендлеры:
- `/start`, `/today`, `/week`, `/digest`, `/search`, `/skills` — **owner only** (is_owner check)
- `/help`, `/info`, `/ideas`, `/rejected`, `/split`, `/classify` — **team+** (is_team_member check)
- Гостевые сообщения — принимаются всегда

Если гость вызывает команду:
```
гость: /info
→ (молчание)
```
Бот НЕ отвечает «нет прав» — просто игнорирует. Это by design (не светить внутреннюю кухню).

## Как добавить новую команду

1. Открой `runtime/scripts/telegram_notes_bot.py`.
2. Найди секцию `# ----- Bot & dispatcher -----` и `dp.message.register(...)` в конце.
3. Создай хендлер:
   ```python
   @dp.message(Command("mycommand"))
   async def cmd_mycommand(message: Message) -> None:
       if not is_owner(message.from_user):
           return
       # ... твоя логика ...
       await message.answer("...")
   ```
4. Добавь в `set_bot_commands()`:
   ```python
   BotCommand(command="mycommand", description="Что делает"),
   ```
5. Если нужны права team+ — используй `is_team_member` вместо `is_owner`.
6. Перезапусти watchdog (kill текущий бот-процесс → watchdog поднимет новый).

## Лимиты и форматирование

- **Telegram message limit:** 4096 символов. Бот автоматически разбивает длинные ответы на чанки (`_split_for_tg`, default 3800 chars на чанк).
- **Markdown/HTML:** бот использует `parse_mode=HTML` по умолчанию. Команды с большим выводом (`/info`, `/ideas`) явно передают `parse_mode=None` для надёжности.
- **Rate limits:** бот НЕ имеет встроенного rate-limit на исходящие. Telegram API имеет свой лимит (30 msg/sec на бота, 1 msg/sec на чат). Для больших рассылок (>30 msg) — пауза между чанками.
