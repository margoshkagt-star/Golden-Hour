# notes-bot-commands

Полный референс всех команд Telegram-бота из Notes-Bot Kit: описание, аргументы, права, примеры вывода.

## Цель

Дать агенту в OpenClaw возможность помочь пользователю бота: подсказать нужную команду, объяснить что она делает, помочь с аргументами.

## Триггер

Когда пользователь:
- Спрашивает «что умеет бот», «какие команды», «что делает /info»
- Хочет посмотреть идеи / заметки / статистику
- Хочет сгенерировать SKILL.md из идей
- Хочет разбить сложную идею на части
- Хочет посмотреть отсеянные (rejected) записи

## Логика

### Все команды (12 штук)

| Команда | Аргументы | Права | Что делает |
|---|---|---|---|
| `/start` | — | owner | Регистрация владельца (chat_id) + приветствие |
| `/help` | — | все | Показывает команды |
| `/today` | — | owner | Заметки за сегодня |
| `/week` | — | owner | Заметки за последние 7 дней |
| `/digest` | — | owner | Непрочитанные с последней рассылки |
| `/search` | `<текст>` | owner | Substring-поиск по content |
| `/info` | `[scope]` или `[mine]` или `[stats]` | team+ | Категоризированная витрина / личная сводка / статистика |
| `/ideas` | `[scope]` | team+ | Список валидных идей (без мусора) |
| `/skills` | `[scope]` | owner | Сгенерировать SKILL.md локально |
| `/rejected` | `[category]` | team+ | Отсеянные идеи |
| `/split` | `<текст идеи>` | team+ | Разбить идею на под-идеи |
| `/classify` | `[new-only]` | team+ | Прогнать intake, обновить ideas.md |

### scope (для /info, /ideas, /skills)

- `all` — по умолчанию
- `today` — только сегодня (по Москве)
- `week` — последние 7 дней
- `month` — последние 30 дней

Пример: `/ideas week`, `/info mine`, `/skills all`.

### category (для /rejected)

- `command` — `/info`, `/start`, etc.
- `test_input` — `ппп`, `тест`, `аааа`
- `media_no_caption` — фото без подписи
- `empty` — пустой content
- `duplicate` — дубль в течение 30 мин
- `manual` — помечены `is_idea=false` руками
- `unknown` — не классифицировано

### RBAC

```python
# telegram_notes_bot.py
def is_owner(user):       # username match (case-insensitive)
def is_team_member(user): # owner OR username в team
```

Гость вызывает команду → бот МОЛЧА игнорирует (не отвечает «нет прав»). Гостевые СООБЩЕНИЯ (не команды) — записываются всегда.

### Примеры вывода

**`/info` (витрина):**
```
📚 Витрина идей команды (все)
📊 27 хороших идей в 8 категориях · 2026-06-16 → 2026-06-23

─── 🤖 Автоматизация и скиллы (5) ───

• «Бот должен уметь подсчитывать повторяющиеся задачи»
  2026-06-17 01:27 · @beatusx · ⭐2
...
✅ Уже реализовано скиллами: 4 из 27
```

**`/ideas week`:**
```
💡 Идеи (за неделю) — 8 шт.

  • 2026-06-23 14:30 — @beatusx 💬: «Идея — присылать мемы по настроению»
  • 2026-06-21 16:04 — @beatusx 💬: «...»
      └─ 1. Бот напоминает про здоровые активности
      └─ 2. После активной сессии решения задач
      └─ 3. Предложить потрогать траву
```

**`/split <идея>`:**
```
🪓 Разбивка идеи — 3 под-идеи:

  Исходная идея:
    «Бот должен понимать настроение, присылать мемы и помогать с CI»

  Под-идеи:
    1. Бот должен понимать настроение
    2. Присылать мемы
    3. Помогать с CI
```

**`/rejected`:**
```
🗑 Отсеянные идеи (13 шт.)

  По категориям:
  • command: 7 (54%)
  • test_input: 3 (23%)
  • manual: 2 (15%)
  • media_no_caption: 1 (8%)

  Использование: /rejected <category>
```

## Вход → Выход

| Вход | Действие | Выход |
|---|---|---|
| «Что умеет бот?» | Список 12 команд с кратким описанием | Таблица из секции «Все команды» |
| «Как посмотреть идеи?» | `/ideas [scope]` | Команда с примером |
| «Как сгенерить SKILL.md?» | `/skills [scope]` или forge-skill агент | Команда + ссылка на pipeline |
| «Что делает /info mine?» | Описание | Личная сводка с датами/важностью |
| «Хочу разбить идею» | `/split <текст>` | Команда + правила сплиттера |
| «Что такое scope?» | Объяснить all/today/week/month | Таблица scope |

## Что НЕ делает

- Не вызывает команды за пользователя (только подсказывает синтаксис)
- Не создаёт новые команды (см. `COMMANDS.md → «Как добавить новую команду»`)
- Не объясняет что делать если команда не работает (это к `notes-bot-operator`)

## Зависимости

- Бот запущен и доступен
- Пользователь аутентифицирован как owner или team member (для соответствующих команд)
- `memory/notes.jsonl` существует и не пустой (для команд чтения)
