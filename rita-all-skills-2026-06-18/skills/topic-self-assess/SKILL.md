# topic-self-assess

## Цель
Проверить **уровень пользователя конкретно по выбранной теме** (`study_topic`). Вопрос адаптируется под тему — не общая самооценка по предмету.

## Триггер
- После `topic-clarify`
- До `setup-finalize`

## Логика
1. Прочитать `users/<user_key>/profile.md` → `study_topic`, `study_subject`, `study_topic_scope`
2. Сформулировать **один вопрос** про эту тему с вариантами ответа
3. При возможности — разбить тему на 2–4 подпункта и спросить по каждому
4. **Записать в `users/<user_key>/profile.md`:**
   - `topic_level: zero|weak|medium|strong|expert`
   - и/или `topic_sublevels: {подтема: уровень}`
   - и/или `topic_level_note: <дословно>`
5. Подтвердить и перейти к `setup-finalize`

## Шкала уровня

| Код | Для пользователя |
|---|---|
| `zero` | **С нуля** |
| `weak` | **Слабо** |
| `medium` | **Средне** |
| `strong` | **Уверенно** |
| `expert` | **Продвинутый / Отлично** |

## Формат сообщения

> 📊 Что уже **знаешь по теме «[study_topic]»**?
>
> 1. **С нуля**
> 2. **Слабо**
> 3. **Средне**
> 4. **Уверенно**
> 5. **Продвинутый**
>
> Или опиши своими словами: что умеешь, что не получается.

## Адаптация под тип темы

**Математика / физика / информатика** — подпункты, если тема широкая:
> По **[study_topic]** что ближе?
> - теория: с нуля / базово / уверенно
> - задачи: не решаю / простые / сложные

**Языки** — про 4 навыка (R/W/L/S) + фонетика:
> - **Чтение (R)**: не читаю / простые тексты / адаптированные / оригинал
> - **Письмо (W)**: не пишу / простые фразы / эссе / статьи
> - **Аудирование (L)**: не понимаю / медленная речь / нормальная / быстрая
> - **Говорение (S)**: не говорю / базовые фразы / диалог / свободно
> - **Произношение/фонетика** (опционально): не владею / базовое / хорошее / носитель
>
> **CEFR mapping** (если цель — конкретный уровень):
> - A1: R/W/L/S все «не владею / не пишу / не понимаю / не говорю»
> - A2: R/W/L/S все «простые тексты / фразы»
> - B1: R/W «адаптированные/эссе», L/S «нормальная/диалог»
> - B2: всё «оригинал/эссе/нормальная/свободно»
> - C1/C2: «статьи / свободно / быстрая»

**Программирование** — про опыт:
> - видел код / писал сам / решал задачи / contest-level

**Проект** (для `purpose: topic` с `study_topic_scope` содержит «проект» / «MVP» / «приложение»):
> Шкала **не «знаю/не знаю»**, а **«умею/не умею»**:
> - 1 (zero) — нет опыта, нужно учить с нуля
> - 2 (weak) — базовое понимание, нужна практика
> - 3 (medium) — могу сделать простую версию (proof of concept)
> - 4 (strong) — могу сделать production-качество
> - 5 (expert) — могу наставничать / архитектурить
>
> По **каждому ключевому навыку/компоненту** проекта — отдельный sublevel. Например, для AI-todo:
> - `react`: strong
> - `uiux`: medium
> - `backend_node`: zero
> - `supabase`: zero
> - `postgres`: weak
> - `llm_api`: zero
> - `vercel_deploy`: weak
> - `env_vars`: medium
>
> Пользователь сам называет список подпунктов — агент не выдумывает.

## Поведение

| Ответ | Куда записать |
|---|---|
| Цифра 1–5 | `topic_level` по шкале |
| «знаю теорию, задачи не решаю» | `topic_sublevels: {"теория": "medium", "задачи": "weak"}` |
| Свободный текст | `topic_level_note` дословно |
| «не знаю» | Уточнить: «С нуля или что-то уже пробовал?» |
