# Приоритеты пользователя — шаблон

> Заполняется при первом запуске `task-triage` на основе 2–3 уточняющих вопросов к пользователю. Далее обновляется автоматически на основе истории.

## Структура

```markdown
## Критичные сферы
- <категория> — <почему критично>
- ...

## Часто откладываемые
- <категория> — <паттерн откладывания>
- ...

## Анти-приоритеты (явно "не горит")
- <категория> — <контекст>
- ...
```

## Пример

```markdown
## Критичные сферы
- 💼 проект:openclaw-assistant — основной рабочий проект, доход
- 💰 финансы — обязательные платежи, штрафы за просрочку

## Часто откладываемые
- 🏥 здоровье — пользователь обычно откладывает визиты к врачу
- 📚 обучение — курсы часто висят без прогресса

## Анти-приоритеты
- 🗄 архив — "почитать когда-нибудь", "посмотреть на досуге"
```
