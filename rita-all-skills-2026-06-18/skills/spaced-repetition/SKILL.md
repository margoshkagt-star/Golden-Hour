# Spaced Repetition — spaced-repetition

**Use when:** в `daily-plan` нужно добавить правило «слабые темы повторяются через увеличивающиеся интервалы», а не каждый день заново.

**Don't use when:**
- Пользователь только начал (нет ещё истории для расчёта интервалов)
- Тема уже `strong` или `expert` (не в фокусе повторений)
- Нужно просто «что повторить сегодня» → `daily-study-checkin`

## Назначение

Вместо плоского «делай X каждый день» — Anki-кривая: **новое → 1 день → 3 → 7 → 14 → 30**. Слабые темы (`zero`/`weak`) автоматически попадают в план на повтор через правильный интервал. Когда тема «закрепилась» (попала в `medium`+) — интервал увеличивается, тема уходит из активного плана.

## Входные данные

Из `users/<user_key>/profile.md`:
- `olympiad_topic_levels` / `exam_topic_levels` / `topic_sublevels` — уровень по темам
- `deadline` — для расчёта «укладываемся ли в повторы до дедлайна»

Из `users/<user_key>/plan.md`:
- список тем с уровнями и фазами
- `progress.md` — история чек-инов, даты последних повторов

## Кривая интервалов (default)

| Уровень сейчас | Интервал повтора | Следующая проверка |
|---|---|---|
| `zero` | **1 день** | завтра |
| `weak` | **3 дня** | через 3 |
| `medium` | **7 дней** | через 7 |
| `strong` | **14 дней** | через 14 |
| `expert` | **30 дней** (или «выходит из ротации») | через 30 |

После каждой проверки:
- **Ответил успешно** → уровень повышается на 1, интервал увеличивается.
- **Ошибся** → уровень понижается на 1, интервал сбрасывается к началу кривой.
- **Не ответил** (пропустил) → помечается `missed`, через 2 дня ещё раз.

## Алгоритм

### 1. Определить «слабые темы»
- Все темы с уровнем `zero` / `weak` / `medium` — кандидаты на повтор.
- Отсортировать по `last_reviewed` ascending (самые «забытые» — первыми).

### 2. Рассчитать due-темы на сегодня
- Для каждой темы: `next_review = last_reviewed + interval(current_level)`.
- Если `next_review <= today` — тема due.
- Ограничить top-N (по умолчанию **3 темы/день**, иначе план раздувается).

### 3. Вставить в `daily-plan` как «review tasks»
- В отличие от обычных задач (новый материал) — review-задачи помечать тегом `[review]`.
- Конкретика: «решить 3 задачи на тему X (уровень Y, последний повтор — N дней назад)».
- **Не прикладывать ссылки на теорию** — только задачи. Теория выдаётся по запросу.

### 4. Обновить state при чек-ине
- В `daily-study-checkin`: спросить «как прошёл повтор по X?», получить качество ответа (1–5).
- Перевести в новый уровень по правилу:
  - качество 1–2 → down на 1 уровень
  - качество 3 → без изменений
  - качество 4–5 → up на 1 уровень
- Записать `last_reviewed = today`.

### 5. Адаптация под дедлайн
- Если `days_until_deadline < 14` и есть `weak` темы, которые не успеют пройти полную кривую → **сжать интервалы**: zero=1, weak=2, medium=4, strong=7.
- Если `days_until_deadline < 7` → **все слабые темы идут в ежедневный review**, без интервалов.

## Анти-паттерны

- ❌ «Повторять каждый день» — Anki-кривая работает именно за счёт **увеличения** интервалов
- ❌ Помечать `expert` темы на повтор через 30 дней, если дедлайн через 10 — бессмысленно
- ❌ Review-задачи в общем списке без тега — теряются среди нового материала
- ❌ «Повторять тему через 1 день» для `medium` — слишком часто, выгорание
- ❌ Спрашивать пользователя «хочешь повтор?» — нет, due-темы идут в план автоматически

## Связанные навыки

- `daily-plan` — потребитель due-списка
- `daily-study-checkin` — обновляет уровни по результатам review
- `study-plan` — источник списка тем с уровнями
- `goal-planned` — в глубоком режиме можно пометить темы как `high-priority-review`

## Метрики успеха (что смотреть через 2–4 недели)

- Средний интервал `medium` тем (если 5–7 дней — кривая работает)
- Доля `weak→medium` переходов за неделю
- «Забытость» — сколько due-тем пропущено (если >30% — что-то не так)
