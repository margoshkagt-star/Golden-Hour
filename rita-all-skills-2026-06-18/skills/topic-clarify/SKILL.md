# topic-clarify

## Цель
Точно понять **какую тему** пользователь хочет изучать — без привязки к экзамену или олимпиаде.

## Триггер
- **Сразу после `purpose-select`**, если `purpose = "topic"`
- **До** `topic-self-assess`

## Логика
1. Спросить тему свободно + предложить структуру ответа
2. Если ответ слишком широкий («математика») — сузить до конкретного блока
3. Если слишком узкий — принять как есть
4. **Записать в `users/<user_key>/profile.md`:**
   - `study_topic: <основная тема, дословно>`
   - опционально `study_subject: <предметная область>`
   - опционально `study_topic_scope: <уточнение границ>`
5. Подтвердить и перейти к `topic-self-assess`

## Формат сообщения

> 🔍 **Какую тему** хочешь изучать?
>
> Напиши как удобно — например:
> - «интегралы в математике»
> - «Present Perfect в английском»
> - «динамическое программирование»
> - «электродинамика, только конденсаторы»
>
> Чем конкретнее — тем точнее подстрою план.

## Уточнение при широком ответе

| Ответ | Реакция |
|---|---|
| `математика`, `физика`, `история` и т.п. | «Уточни: алгебра / геометрия / теория чисел / матанализ / что-то конкретное?» |
| `программирование` | «Какое направление: алгоритмы / веб / данные / ML?» |
| `английский` | «Какой аспект: грамматика / лексика / разговорная речь?» |
| **Продуктовый/творческий проект** («сделать MVP», «приложение», «сервис», «книга», «курс») | ⚠️ **Это НЕ learning-цель** — адаптация в `topic-self-assess` не подойдёт. См. ветку «Проект» ниже. |
| Конкретная тема | Принять как `study_topic` |

## Ветка «Проект» (`study_topic_scope` содержит «проект» / «MVP» / «приложение» / «сервис»)

Для продуктовых/творческих целей:
1. **Принять `study_topic`** как название проекта (напр. «MVP AI-todo», «книга по физике»).
2. **`study_subject`** = область (напр. «fullstack web», «creative writing», «edtech»).
3. **`study_topic_scope`** = желаемый результат + сроки (напр. «MVP за 2 месяца, деплой на Vercel»).
4. **`topic_sublevels`** — ключевые навыки/компоненты проекта (backend, frontend, deploy и т.п.) — **пользователь сам называет**, не из шаблона.
5. **Прямая рекомендация:** «Этот проект — не учебная тема, а результат. Скилл `topic-self-assess` оценивает уровень по теме, а не навыки. Если хочешь оценку по навыкам проекта — продолжим через `topic-self-assess` (ветка «Проект»). Если хочешь сразу план с фазами/milestones — переходим в `goal-planned` с `purpose: project`».

**Важно:** в текущей версии `purpose` принимает значения `exam` / `olympiad` / `topic`. Для проектов рекомендую использовать `purpose: topic` с пометкой `study_topic_scope: "проект"`, либо в будущем добавить `purpose: project` в `purpose-select`.

- Перед `topic-self-assess`
