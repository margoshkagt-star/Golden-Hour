# task-decomposition

## Цель
Автоматически разбивать крупные задачи (вес ≥ 4, оценка > 1 дня) на атомарные подзадачи по 15-45 минут, согласно правилам существующего `task-triage` скилла в golden-hour.

## Триггер
- **Авто:** при добавлении задачи в `users/<user_key>/tasks.md` с `weight ≥ 4` и `estimate > 1d`.
- **Вручную:** команда «разбей задачу X на шаги» / «декомпозируй Y».

## Логика
1. Прочитать задачу из `users/<user_key>/tasks.md` (формат: `### id: t_NNN` / `- title: <строка>` / `- weight: 1-5` / `- estimate: 15m/30m/1h/2h/1d+`).
2. Если `weight < 4` или `estimate ≤ 1h` — стоп, не декомпозировать.
3. Прочитать `users/<user_key>/plan.md` — текущий макро-план (недели), чтобы новые подзадачи вписались.
4. Сгенерировать 3-7 подзадач по правилам:
   - Каждая ≤ 45 минут.
   - Каждая имеет проверяемый результат (что значит «готово»).
   - Первая подзадача ≤ 15 минут (быстрый старт).
   - Последняя — интеграция/проверка результата.
5. Записать подзадачи в `users/<user_key>/tasks.md` под оригинальной задачей как вложенные (или отдельным блоком).
6. Обновить `progress.md` — добавить запись «декомпозировал задачу t_NNN на N подзадач (t_NNN.1 — t_NNN.N)».
7. Сообщить пользователю: «Разбил t_NNN «X» на N подзадач: …».

## Вход / Выход
- **Вход:** `task_id` (например `t_007`), контекст пользователя.
- **Выход:** N подзадач в `tasks.md` + запись в `progress.md` + сообщение в чате.

## Примеры
### Пример 1
**Запрос:** «разбей задачу t_007 “Подготовить оферту” на шаги»
**Результат:**
- t_007.1 — собрать требования (15m)
- t_007.2 — изучить шаблоны (30m)
- t_007.3 — черновик структуры (30m)
- t_007.4 — написать разделы 1-3 (45m)
- t_007.5 — ревью с юристом (30m)
- t_007.6 — финальная версия (15m)
(всего 6 подзадач, общая оценка 2h 45m)

## Что НЕ делает
- ❌ Не декомпозирует задачи с оценкой меньше 1 часа.
- ❌ Не переписывает уже декомпозированные задачи (idempotency: проверяет наличие `t_NNN.*` в `tasks.md`).
- ❌ Не меняет существующую задачу — только добавляет подзадачи.
- ❌ Не работает с голосовыми и фото без текстовой расшифровки.
