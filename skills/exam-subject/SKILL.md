# exam-subject

## Цель
Уточнить **предмет экзамена**. Без предмета — не продолжать.

## Триггер
- После `exam-type`
- До `exam-topics`

## Логика
1. Показать список предметов (адаптировать под `exam_type`, если очевидно)
2. Получить ответ
3. **Записать в `users/<user_key>/profile.md` → `exam_subject: <код>`**
4. Если «другой» — `exam_subject_note: <текст>`
5. Подтвердить и перейти к `exam-topics`

## Формат сообщения

> 📚 Какой **предмет** на экзамене?
>
> 1. **Математика** (база / профиль — уточним позже)
> 2. **Русский язык**
> 3. **Физика**
> 4. **Химия**
> 5. **Биология**
> 6. **Информатика**
> 7. **Обществознание**
> 8. **История**
> 9. **Английский**
> 10. **Другой** (напишу какой)
>
> Цифра или слово — что угодно.

## Поведение

| Ответ | exam_subject |
|---|---|
| `1` / `математика` | `math` |
| `2` / `русский` | `russian` |
| `3` / `физика` | `physics` |
| `4` / `химия` | `chemistry` |
| `5` / `биология` | `biology` |
| `6` / `информатика` | `informatics` |
| `7` / `обществознание` | `social` |
| `8` / `история` | `history` |
| `9` / `английский` | `english` |
| `10` / `другой` / свой текст | `other` + `exam_subject_note` |

**ЕГЭ математика:** если пользователь сразу пишет «профиль» / «база» — записать `exam_subject_variant: profile|base`.

## Данные
- `users/<user_key>/profile.md` → `exam_subject`, опционально `exam_subject_note`, `exam_subject_variant`

## Зависимости
- Зависит от `exam-type`
- Перед `exam-topics`

## Где живёт реальное исполнение

**`SOUL.md` → секция «Старт сессии» → экзаменационная ветка (шаг 11)** — это то, что агент реально выполняет. Этот скилл — **дизайн-документ** для справки и эволюции.
