# Повторяющиеся задачи (recurring-tasks)

> Идея @beatusx (важность 2, тема «автоматизация задач»), доведена до рабочего скилла.

Позволяет один раз описать регулярное дело («каждый день 30 мин английский», «по вт/чт — разбор задач»), и дальше оно само появляется в дневном плане без переспрашивания.

**Хранение:** `users/<user_key>/recurring.json`. Требует `setup_status: complete`.

## Когда использовать
- Пользователь говорит «делай это каждый день / по будням / раз в неделю».
- `daily-plan` собирает день — подмешать актуальные на сегодня recurring-задачи.
- Пользователь просит «покажи/измени/выключи повторяющиеся задачи».

## Формат правила
`users/<user_key>/recurring.json`:
```json
{
  "schema_version": 1,
  "rules": [
    {
      "id": "r_eng_daily",
      "title": "Английский — карточки",
      "freq": "daily",            // daily | weekly | weekdays | custom
      "days": [],                 // для weekly/custom: [1..7], Пн=1
      "duration_min": 30,
      "weight": 3,                // для task-weighting
      "difficulty": 2,
      "active": true,
      "skip_dates": ["2026-06-20"],
      "last_done": "2026-06-18"
    }
  ]
}
```

## Логика
1. **Создать/изменить** правило по запросу (распарсить частоту и длительность).
2. **Раскрытие на дату** (вызывает `daily-plan`): для каждой `active`-задачи проверить, попадает ли `date` под `freq`/`days` и не в `skip_dates`. Если да — добавить как задачу с тегом `[recurring]`, передать `weight`/`difficulty` в `task-weighting`/`daily-balancer`.
3. **Учёт выполнения:** при чек-ине (`daily-study-checkin`) проставить `last_done`; пропуск → не ломает streak других задач.
4. **Связь с интервалами:** для тем на повтор используется `spaced-repetition` (растущие интервалы), а recurring — для фиксированных бытовых дел. Не дублировать одну тему в обоих.

## Что НЕ делает
- ❌ Не плодит копии задачи на каждый прошедший день (раскрытие — только на запрошенную дату).
- ❌ Не заменяет `spaced-repetition` для слабых тем.
- ❌ Не работает до завершения настройки.

## Связанные навыки
- `daily-plan` — потребитель раскрытых на сегодня правил.
- `task-weighting` / `daily-balancer` — вес и раскладка.
- `spaced-repetition` — для тем с растущими интервалами (а не фикс. частотой).
- `organizaciya-povtora-deistvii` — общий «учёт повторяющихся действий».
