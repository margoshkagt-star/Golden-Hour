# Reflection Loop — reflection-loop

**Use when:** пользователь сообщает «не успел», «провалил», «сорвал дедлайн», ИЛИ детектор пропуска фиксирует `missed` 2+ раза подряд.

**Don't use when:**
- Просто отстали на 1 день — это нормальный drift, адаптация через `daily-plan`
- Пользователь сам понимает, что произошло, и просит сдвинуть дедлайн — тогда сразу пересборка через `study-plan`
- Разовое «не сделал Х» (может быть, приоритет не тот был) — не включать рефлексию, просто перенести

## Назначение

Превратить провал в обучающий момент, а не в самобичевание. Три цели:
1. **Найти причину** (системную, не «я ленивый»)
2. **Зафиксировать** в `users/<user_key>/progress.md` (паттерн, не разовый провал)
3. **Адаптировать план** (что менять, чтобы не повторилось)

## Триггеры (любой из)

| Триггер | Источник |
|---|---|
| «Не успел / сорвал / провалил» | явный репорт пользователя |
| `missed_count >= 2` для одной темы/задачи | `task-tracker` детектор |
| Дедлайн milestone прошёл, прогресс < 70% | `study-plan` проверка |
| 3+ дня streak потерь подряд | `daily-study-checkin` |

## Логика

### 1. Пауза и факт
- Не начинать с «ничего страшного». Сразу факт: «Тема X должна была быть закрыта Y дней назад, сейчас прогресс Z%».
- **Без осуждения.** Без «ну что ж ты». Без «давай не будем расстраиваться».

### 2. Один вопрос — «почему?» — с вариантами
Не открытый «что случилось?», а **меню причин** (на основе частых паттернов):

```
Что помешало? (можно несколько)
1. Не хватило времени — план был нереалистичный
2. Не хватило сил/фокуса — было тяжело
3. Не понял тему — нужен другой источник/объяснение
4. Были срочные другие задачи — приоритет не тот
5. Отвлёкся на другое — забил
6. Забыл про дедлайн — не было напоминаний
7. Болел / форс-мажор
8. Свой вариант: ___
```

Пользователь может выбрать несколько или написать своё. **Каждый ответ — отдельная строка в `users/<user_key>/progress.md`**, не один блок.

### 3. Анализ системы
После получения ответов — посмотреть, есть ли **системная причина** (повторяется):

- **«Не хватило времени»** 2+ раза подряд → план нереалистичен, пересмотр бюджета
- **«Не понял тему»** 2+ раза по одной теме → заменить источник, разбить на подтемы
- **«Отвлёкся»** 3+ раза → попросить включить в `goal-checkin-notifier` напоминания или выключить отвлекающие
- **«Забыл»** → усилить `goal-checkin-notifier`, пересмотреть время напоминаний

### 4. Адаптация (конкретные действия)

| Причина | Действие |
|---|---|
| План нереалистичный | `study-plan` пересборка: меньше часов/нед, больше недель; снизить `daily_load` |
| Тема непонятна | Разбить на подтемы (`ideya-razbivat-krupnye-zadachi-na`/`task-triage`), запросить материалы |
| Приоритет не тот | `task-weighting`: сравнить вес задачи с текущим фокусом, переставить |
| Отвлечение | Включить `goal-checkin-notifier` morning/evening, dry-mode |
| Забыл дедлайн | Добавить milestone reminder за 3 дня |

### 5. Запись в `users/<user_key>/progress.md`
```markdown
### YYYY-MM-DD — рефлексия после [провал/просрочка]
- **Что:** <тема/задача>
- **Дедлайн был:** YYYY-MM-DD
- **Прогресс:** X%
- **Причины (выбрано):** 1, 4
- **Системная причина:** да — план нереалистичный + неправильный приоритет
- **Адаптация:** пересобрал plan.md, бюджет 4→3 ч/нед, переставил milestone
- **Урок:** не оценивать тему по «давай попробую», сначала прикинуть объём
```

### 5b. Запись в temporal-kg (обязательно после progress.md)
```bash
node scripts/temporal-kg.mjs reflection --user <user_key> \
  --topic "<тема>" --what "<что>" --causes "<причины>" --adaptation "<что меняем>"
```
При просрочке — дополнительно:
```bash
node scripts/temporal-kg.mjs drift --user <user_key> --topic "<тема>" --days-late N --reason "<кратко>"
```

### 6. Forward commit
- **Один** новый реалистичный micro-commit на ближайшие 2 дня (не «доделать всё»).
- Срок: пересмотр через 48 ч.

## Анти-паттерны

- ❌ «Ничего страшного, бывает» — фальшивое утешение, не работает
- ❌ «Ты молодец, что старался» — снисходительно, не помогает
- ❌ Открытый «что случилось?» — размывает фокус
- ❌ Сразу прыгать в «как починим» без фиксации причины
- ❌ Наказывать (отнимать время от других тем) — не рефлексия, а месть
- ❌ Делать рефлексию по мелочам (1 пропущенный день) — выгорание
- ❌ Не записать в `progress.md` — через 2 недели паттерн забудется

## Связанные навыки

- `study-plan` / `daily-balancer` — адаптация плана и бюджета после рефлексии
- `daily-study-checkin` — может триггерить рефлексию, если streak потерян
- `task-tracker` — детектор `missed_count`
- `users/<user_key>/progress.md` — хранилище причин и уроков

## Тон

Прямой, по делу, без терапевтического подтекста. Не «давай поговорим о твоих чувствах», а «что сломалось, как чинить». В духе: «Я знаю, что ты умная и знаешь, что делать. Давай зафиксируем, чтобы паттерн не повторился».
