# legal-offer-draft

## Цель
Помогает пользователю создать черновик оферты (legal offer) для нового сервиса по структурированному шаблону. Запрашивает недостающие поля (юр. лицо, предмет, цена, сроки) и формирует документ.

## Триггер
- **Вручную:** команда «подготовь оферту для X» / «черновик оферты».

## Логика
1. Спросить пользователя:
   - Название сервиса
   - Юр. лицо (ООО / ИП / самозанятый, ИНН)
   - Предмет оферты (что продаёте / оказываете)
   - Цена / тарифы
   - Срок действия
   - Способ оплаты
   - Контакты
   - Особые условия (если есть)
2. Подтянуть шаблон из `references/offer-template.md` (стандартная структура: преамбула, предмет, цена, сроки, оплата, ответственность, реквизиты).
3. Подставить данные пользователя в шаблон.
4. Сгенерировать markdown-документ.
5. Сохранить в `users/<user_key>/documents/offer-<service>-<date>.md` (новый файл).
6. Сообщить пользователю: «Черновик оферты готов: <путь>. Проверь с юристом перед публикацией».

## Вход / Выход
- **Вход:** данные о сервисе (7 полей).
- **Выход:** markdown-файл оферты.

## Примеры
### Пример 1
**Запрос:** «подготовь оферту для сервиса консультаций по AI, 5000 руб/час, ООО "Ромашка"»
**Результат:** `documents/offer-ai-consulting-2026-06-16.md` с заполненными секциями.

## Что НЕ делает
- ❌ Не заменяет юриста — только шаблон с подставленными данными.
- ❌ Не публикует оферту (только создаёт файл).
- ❌ Не делает ЭЦП / не отправляет в налоговую.
- ❌ Не работает с многостраничными приложениями (один документ = одна оферта).

## Зависимости
- skills: `note-to-file`, `user-profile`
- tools: `read`, `write`
- данные: `references/offer-template.md` (нужно создать), `users/<user_key>/documents/`
