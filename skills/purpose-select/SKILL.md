# purpose-select

## Цель
Сразу после получения имени — **дать пользователю выбор цели** работы с ботом. Без выбора — не продолжать.

## Триггер
- Срабатывает **сразу после `hello-intro`** (имя получено и записано)
- **До** ветки настройки и `setup-finalize`

## Логика
1. Показать 3 варианта цели:
   - **Экзамен** — ЕГЭ / ОГЭ / вступительные / другой формальный экзамен
   - **Олимпиада** — ВсОШ / перечневая / другая предметная олимпиада
   - **Тема** — изучаю конкретную тему/предмет без привязки к дедлайну
2. Получить выбор пользователя (любая форма: цифра, слово, развёрнуто)
3. **Записать в `users/<user_key>/profile.md`** (поле `purpose`: `exam` / `olympiad` / `topic`)
4. Подтвердить запись
5. **Ветвление по цели** — уточнить, что пользователь уже знает (отдельные скиллы):
   - `olympiad` → `olympiad-grade` → `olympiad-subject` → `olympiad-self-asses`
   - `exam` → `exam-type` → `exam-subject` → `exam-topics` → `exam-self-assess`
   - `topic` → `topic-clarify` → `topic-self-assess`

## Формат сообщения

> 🌅 Записал — **[name]**. Отлично!
>
> Теперь выбери, **зачем тебе бот**:
>
> 1. **Экзамен** — ЕГЭ / ОГЭ / вступительные / другой формальный экзамен
> 2. **Олимпиада** — ВсОШ / перечневая / другая предметная олимпиада
> 3. **Тема** — изучаю конкретную тему/предмет без дедлайна
>
> Напиши цифру или слово (`1` / `2` / `3` / `экзамен` / `олимпиада` / `тема`) — что угодно.

## Поведение при разных ответах

| Ответ | purpose | Реакция |
|---|---|---|
| `1` / `экзамен` / `егэ` / `огэ` / `экз` | `exam` | `Записал: экзамен.` |
| `2` / `олимпиада` / `олимп` / `всош` / `перечнев` | `olympiad` | `Записал: олимпиада.` |
| `3` / `тема` / `изучение` / `предмет` | `topic` | `Записал: тема.` |
| `не знаю` | — | Уточнить: «экзамен с дедлайном, олимпиада с дедлайном, или просто тема без дедлайна?» |
| Другое | — | Переспросить, не осуждать |

## Что НЕ делает
- **Не переходит к следующим шагам**, пока цель не выбрана
- Не оценивает выбор («олимпиада круче» — никогда)
- Не выбирает за пользователя
- Не навязывает вариант
- Не выбирает `purpose` за пользователя (загрузку старого профиля решает `session-start`)

## Стиль
- Тёплый, без давления
- Нейтральный ко всем трём вариантам
- Без «что тебе важнее» — просто перечисление

## Кастомизация
- Список вариантов — редактируется через `revise`
- Можно добавить 4-й вариант (например, «прокачаться в чём-то»)
- Поле `purpose` в `users/<user_key>/profile.md` — для дальнейшей персонализации напоминаний и чек-инов

## Данные
- `users/<user_key>/profile.md` → поле `purpose` (`exam` / `olympiad` / `topic`)

## Зависимости
- **Зависит от `hello-intro`** — имя должно быть получено
- **Перед** веткой уточнения знаний (`olympiad-*` / `exam-*` / `topic-*`)
- **Перед** веткой настройки → `setup-finalize`

## Где живёт реальное исполнение

**`SOUL.md` → секция «Старт сессии» (шаги 5–8)** — это то, что агент реально выполняет в каждой сессии. Этот скилл — **дизайн-документ** для справки и эволюции.
