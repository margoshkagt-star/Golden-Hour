# Конспект урока из аудио (lesson-notes-from-audio)

> Идея @beatusx, доведена до рабочего скилла.

Пользователь присылает голосовое/аудиозапись урока — скилл делает из неё аккуратный конспект и складывает в библиотеку материалов пользователя.

**Хранение:** `users/<user_key>/materials/<goal_id>/notes/YYYY-MM-DD_<slug>.md`. Требует `setup_status: complete`.

## Зависимость от возможностей среды
Скиллу нужна **транскрипция аудио** (speech-to-text). Перед работой проверить:
- есть ли в окружении инструмент транскрипции (whisper/STT в `AGENTS.md`/`openclaw.json`);
- если транскрипции нет — честно сказать пользователю «не могу расшифровать аудио в этой конфигурации» и предложить прислать текст/фото конспекта вручную. **Не выдумывать содержание записи.**

## Логика
1. **Принять аудио**, определить цель/предмет (из `profile.md` или спросить).
2. **Транскрибировать** в текст (если доступно).
3. **Структурировать** конспект:
   ```markdown
   # <Тема урока> — YYYY-MM-DD
   ## Главное (3–5 тезисов)
   ## Определения / термины
   ## Формулы / правила
   ## Примеры / разборы
   ## Что повторить (→ spaced-repetition)
   ## Вопросы, оставшиеся без ответа
   ```
4. **Сохранить** через `goal-materials` (`type: note`, теги темы), обновить `index.json`.
5. **Предложить** добавить слабые места в `spaced-repetition` и/или задачу-повтор в дневной план.

## Что НЕ делает
- ❌ Не галлюцинирует содержание, если расшифровка недоступна/неразборчива — помечает `[неразборчиво]`.
- ❌ Не хранит сырое аудио дольше, чем нужно для расшифровки (только конспект).
- ❌ Не пишет в чужую папку.

## Связанные навыки
- `goal-materials` — хранилище конспектов.
- `material-cache` — если по теме нужен дополнительный разбор.
- `spaced-repetition` — повтор выявленных слабых мест.
