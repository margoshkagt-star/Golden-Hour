# Deep Web Research (Perplexity-style)

Процедура глубокого web-исследования с многоисточниковым синтезом и цитатами. Заменяет «погуглил один раз и выдал ответ» на полноценный research-агентский подход.

## Когда применять

**Триггеры** (description загружается автоматически):
- «Погугли», «найди в сети», «что нового по X», «сделай research», «разбери тему»
- Вопрос требует свежей инфы из сети (не общие знания)
- Нужны цитаты/источники для отчёта
- Perplexity-style синтез с citations

**Не применять**:
- Локальные вопросы (не про сеть)
- Общие знания («что такое Python?») — web_search тут не поможет
- Критичные по точности данные (юридические, медицинские) — нужна верификация человеком
- Когда Карим явно просит «быстро, одной строкой» — не раздувать research на 5 мин

## Процедура

### Шаг 1. Уточни scope (опционально, max 1 вопрос)
- Без ответа на уточнение задача **непонятна** — задай
- Если scope ясен из контекста — пропусти
- Пример: «Какие аспекты X интересуют — новости за неделю или история вопроса?»

### Шаг 2. Выбери режим

| Режим | Время | web_search | web_fetch | Когда |
|---|---|---|---|---|
| **Быстрый** | 30-60 сек | 1-3 | 0-2 | Простой вопрос, нужна 1-2 ссылки |
| **Глубокий** | 2-5 мин | 3-7 | 3-5 | Стандартный research, citations обязательны |
| **Exhaustive** | 5-15 мин | 5-15 | 5-10 | Академический/маркетинговый разбор, нужна полнота |

**Дефолт**: глубокий. Быстрый — только если Карим явно просил «быстро».

### Шаг 3. Многоходовый поиск

- **2-4 разных формулировки** запроса (англ + рус, разные angles)
- Прогрессия запросов:
  1. Обзор темы («что такое X», «X explained»)
  2. Свежие новости («X news 2026», «X update»)
  3. Мнения/критика («X problems», «X review»)
  4. Данные/статистика («X statistics», «X market size»)
- Используй `date_after` для свежих тем
- Фильтруй домены через `domain_filter` если нужно: `["arxiv.org","github.com"]` для tech, `["reuters.com","bbc.com"]` для новостей
- `country/language` — для локализации (по умолчанию `ru,en`)

### Шаг 4. Извлечение и верификация

- `web_fetch` топ-3-5 результатов (НЕ все подряд — выбирай релевантные)
- Сверяй: даты, источники, факты
- Если источники **противоречат** — отметь это явно в ответе
- Если источник **устарел** (старше 2 лет для tech) — ищи свежее
- Если источник **сомнительный** (blog без автора, форум) — отметь, ищи подтверждение

### Шаг 5. Синтез

**Структура ответа** (обязательно):
1. **TL;DR** (1-3 предложения) — суть в одной фразе
2. **Разбор** — основные тезисы с цитатами
3. **Противоречия** (если есть) — что источники говорят по-разному
4. **Источники** — список URLs с кратким описанием

**Цитирование**:
- Формат: «По данным [название источника](url), ...»
- Или: «[X сообщает](url), что ...»
- Или: «Согласно [отчёту Y](url), ...»
- НЕ выдавай мнение одного источника за факт

**Голос**:
- Нейтральный, аналитический
- Если тема спорная — отметь разные позиции
- Если есть «общепринятый ответ» — скажи это явно

### Шаг 6. Формат вывода

- **На русском** (если Карим не просил иначе)
- **Telegram-friendly**: bullets, заголовки, эмодзи
- **Без markdown-таблиц** (Telegram их плохо рендерит)
- **Цитаты в скобках** с URL
- **В конце блока**: «Источники:» + список URLs

### Шаг 7. Прозрачность по токенам (обязательно)

В конце ответа — короткая мета-строка:
```
⚙️ Режим: глубокий · 5 запросов · 3 fetch · ~12k токенов
```

Это даёт Кариму понять, сколько ресурсов ушло.

## Пример

**Запрос Карима**: «Что нового в AI-планировщиках в 2026?»

**Действия**:
1. Scope ясен (новое за период)
2. Режим: глубокий
3. Запросы: «AI scheduler news 2026», «AI task management trends 2026», «agentic task management 2026», «новое в AI-планировщиках 2026»
4. Fetch: топ-5 результатов
5. Синтез: тренды, лидеры, новинки, цитаты
6. Формат: TL;DR + bullets + ссылки

**Ответ** (сокращённо):
```
🔍 AI-планировщики 2026 — обзор

TL;DR: рынок ушёл в agentic + conversational AI...

🔥 Топ-тренды:
- Agentic PMO (по [статье Asana](url))
- Energy-aware scheduling (по [Lifestack](url))
- ...

⚙️ Режим: глубокий · 5 запросов · 3 fetch · ~12k токенов
```

## Инструменты

- `web_search` — основной
- `web_fetch` — извлечение топ-страниц
- `write` — сохранение длинного отчёта в файл (опционально, для exhaustive)

## Граничные случаи

- **Web_search вернул 0 результатов** → попробуй другие формулировки, если всё равно 0 — скажи Кариму прямо
- **Все источники с одного домена** → добавь предупреждение «источники могут быть предвзяты»
- **Спорная тема** (политика, здоровье) → явно отметь разные позиции, не навязывай одну
- **Карим просил «только русские источники»** → используй `country: "ru"` и `language: "ru"`

## Стоимость (ориентир)

- Быстрый: 2-5k токенов
- Глубокий: 8-20k токенов
- Exhaustive: 25-50k токенов

**Дефолт для Карима**: глубокий (оптимальный баланс).
