# olympiad-self-asses

## Цель
Оценить, **что пользователь уже знает** по выбранному предмету и **на каком уровне**. Вопросы и варианты ответов **адаптируются под `olympiad_subject`** из `users/<user_key>/profile.md`.

## Триггер
- После `olympiad-subject`
- До `setup-finalize`

## Логика
1. Прочитать `users/<user_key>/profile.md` → `olympiad_subject`
2. Выбрать карту тем и уровней для этого предмета (см. ниже)
3. Показать **один** вопрос с вариантами (общий уровень + опционально по темам)
4. Получить ответ
5. **Записать в `users/<user_key>/profile.md`** (общий уровень / по темам / свободный текст — дословно)
6. Подтвердить запись
7. Перейти к `setup-finalize`

## Поведение

| Ответ | Куда записать |
|---|---|
| `1` / `новичок` | `olympiad_level: novice` |
| `2` / `средний` | `olympiad_level: medium` |
| `3` / `продвинутый` | `olympiad_level: advanced` |
| `4` / `топ` | `olympiad_level: top` |
| Цифра темы + уровень («6 — продвинутый») | `olympiad_levels: {"геометрия": "advanced", ...}` |
| Несколько тем сразу | `olympiad_levels: {тема: уровень, ...}` |
| Свободный текст | `olympiad_level_note: <дословно>` |
| «Не знаю» / «сложно оценить» | Уточнить: «Ок, тогда хотя бы: школьная программа / уже решал олимпиадные / есть диплом?» |

## Что НЕ делает
- Не приукрашивает и не занижает уровень
- Не сравнивает с другими («для 8 класса это мало» — никогда)
- Не переходит дальше, пока не получен ответ (хотя бы приблизительный)
- Не показывает карту «не того» предмета — всегда читать `olympiad_subject` из `users/<user_key>/profile.md`

## Данные
- `users/<user_key>/profile.md` → одно или несколько:
  - `olympiad_level: novice|medium|advanced|top`
  - `olympiad_levels: {тема: уровень, ...}`
  - `olympiad_level_note: <текст>`

## Зависимости
