# daily-balancer

Превращает список взвешенных тем (от `task-weighting`) в **сбалансированный день**: приоритет решает порядок и время, сложность — нагрузку.

## Вход
Список кандидатов на сегодня с `{title, eff_priority, eff_difficulty, est_minutes, slot_pref?}` и бюджет дня `D_max` (из `task-weighting`).

## Алгоритм сборки

1. **Сортировать кандидатов по `eff_priority` (убыв.).** При равенстве — слабые/срочные выше.
2. **Набирать в день, пока сумма `eff_difficulty` ≤ `D_max`.**
   - Не больше **одного** блока сложности 5 в день.
   - Если уже есть блок ≥4 — следующий тяжёлый (≥4) не ставить подряд: вставить между ними лёгкий (≤2) блок (повторение/карточки).
   - Если в дне есть тяжёлый блок (≥4), а лёгких нет — добавить один лёгкий, даже мелкий, для разгрузки.
3. **Время по приоритету:** более важным темам — больше минут (в пределах часов дня). Делить `est_minutes` так, чтобы важное получило больше.
4. **Размещение по слотам (когда свежесть важна):**
   - **Утро (10:00)** — самые тяжёлые блоки (≥4) и новая теория.
   - **День (14:00)** — средние (3) и практика/задачи.
   - **Вечер (18:00)** — лёгкие (≤2): повторение, карточки, тест.
5. **Перелив:** что не влезло в бюджет/часы — перенести на ближайший свободный день (отметить «перенесено»), не утрамбовывать.

## Правила «день не слишком сложный»
- Σ `eff_difficulty` ≤ `D_max`.
- Максимум один блок 5/день; максимум два блока ≥4/день и только через лёгкий между ними.
- Хотя бы один лёгкий блок, если день содержит тяжёлый.
- Если все важные темы тяжёлые и не влезают — взять 1–2 самые приоритетные сегодня, остальное завтра (лучше меньше, но выполнимо).

## Правила «важных задач больше»
- Порядок: высокий приоритет — раньше в дне и раньше в неделе.
- Время: доля минут пропорциональна приоритету.
- Частота: приоритет 5 может появляться чаще (несколько дней подряд), приоритет 1–2 — по остаточному принципу.

## Выход
Готовый набор блоков на день: `{title, scheduled_at, est_minutes, eff_priority, eff_difficulty, slot}` → отдаётся в `daily-plan` для записи в `plans/YYYY-MM-DD.json` (поля `weight`, `difficulty` сохраняются в задачах).

## Данные
- Вход: результат `task-weighting` + `D_max`.
- Не пишет сам — пишет `daily-plan`.

## Зависимости
- После `task-weighting`. Вызывается из `daily-plan`.
