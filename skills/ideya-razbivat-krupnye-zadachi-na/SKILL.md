# Разбиение крупных задач (task-breakdown)

> Идея @beatusx (важность 2, тема «Управление задачами»), доведена до рабочего скилла.

Берёт большую/расплывчатую задачу («подготовиться к части C», «выучить стереометрию») и режет её на конкретные подзадачи так, чтобы первый шаг занимал **меньше 15 минут** и было понятно, что делать прямо сейчас.

**Хранение:** `users/<user_key>/tasks.md`. Требует `setup_status: complete`.

## Когда использовать
- Пользователь называет задачу, которую «страшно начать» / непонятно с чего начать.
- В `daily-plan`/`study-plan` появился крупный пункт (> ~90 мин или абстрактный) — авто-разбить.
- Пользователь просит «разбей задачу X».

## Когда НЕ использовать
- Задача уже атомарная (< 25 мин, понятный первый шаг).
- Нужна долгосрочная декомпозиция целой цели по неделям → это `study-plan`.

## Логика
1. **Уточнить рамки** (1 вопрос, если непонятно): срок, объём, известный уровень (из `profile.md`).
2. **Разбить** на 3–7 подзадач по принципу:
   - каждая — глагол + конкретный объект («решить 5 задач №16 на параметры»),
   - первая подзадача — «разогрев» < 15 мин,
   - порядок — от простого к сложному.
3. **Оценить** каждую через `task-weighting` (`difficulty` 1–5, `weight` 1–5) — чтобы `daily-balancer` мог корректно разложить по дням.
4. **Записать** в `users/<user_key>/tasks.md`:
   ```markdown
   ## <крупная задача> — разбита YYYY-MM-DD
   - [ ] (1) <шаг <15 мин>  · w2 d1
   - [ ] (2) <шаг>          · w3 d3
   - [ ] (3) <шаг>          · w3 d4
   ```
5. **Предложить** добавить первую 1–2 подзадачи в дневной план сегодня.

## Что НЕ делает
- ❌ Не плодит микрозадачи (> 7 — признак, что нужна декомпозиция цели в `study-plan`).
- ❌ Не пишет в чужую папку.
- ❌ Не оставляет первый шаг тяжёлым.

## Связанные навыки
- `task-triage` — общий движок декомпозиции/категоризации (можно переиспользовать).
- `task-weighting` / `daily-balancer` — оценка и раскладка подзадач.
- `study-plan` — декомпозиция уровня цели (не отдельной задачи).
