# study-plan

Строит **долгосрочный план подготовки** под цель пользователя и сохраняет в его папку. Источник правды для `daily-plan` и чек-инов.

## Триггер
- Авто: из `setup-finalize` сразу после завершения настройки
- Вручную: «составь план», «пересобери план», «обнови план», «план на год»
- Требует `setup_status: complete`

## Логика

1. Прочитать `users/<user_key>/profile.md`: `purpose`, ветку, уровни, `deadline`, `hours_per_week`.
2. **Посчитать бюджет:**
   - Недель до дедлайна = (дедлайн − сегодня) в неделях. Без дедлайна → горизонт 12 недель по умолчанию.
   - Всего часов = `hours_per_week × недель`.
   - Разложить часы по темам через `task-weighting`: **больше времени — темам с высоким `eff_priority`** (важность × близость дедлайна × слабость). Слабые и важные — раньше и плотнее.
3. **Собрать темы** под ветку и отсортировать по `eff_priority` (убыв.):
   - olympiad → темы предмета из `olympiad_levels`
   - exam → `exam_topics` с уровнями из `exam_topic_levels`
   - topic → подпункты `study_topic` / `topic_sublevels`
   - В каждой неделе помечать ожидаемую сложность тем (для `daily-balancer`).
4. **Сгенерировать план** по шаблону (недельный скелет + бюджет + чек-поинты).
5. Сохранить в `users/<user_key>/plan.md`. Обновить `profile.md` → `updated`.
6. Показать сводку (не весь план): сколько недель, часов, порядок тем, ближайшая неделя.

## Шаблон `plan.md`

```markdown
# План подготовки — <name>

> Создан: YYYY-MM-DD HH:MM
> Цель: <purpose> — <subject/topic> · Дедлайн: <deadline> · <hours_per_week> ч/нед

## Бюджет
- Недель до дедлайна: <N>
- Всего часов: <hours_per_week × N>
- Распределение: <тема: часы> (слабым — больше)

## Темы (порядок: слабые → сильные)
| # | Тема | Уровень | Часов | Недели |
|---|---|---|---|---|
| 1 | ... | с нуля | ~X | 1–k |

## Понедельный скелет
1. **Нед. 1** — <тема>: <что делать>
2. **Нед. 2** — ...
...

## Чек-поинты
- По итогу каждой темы — короткий тест 5–10 заданий
- Раз в ~4 недели — пробный вариант
- Недельный ритм: <hours_per_week> ч

## Следующий шаг
Сказать «неделя 1» — выдам задание по первой теме.
```

## Адаптация под цель
- **olympiad** — упор на разбор сложных задач по слабым темам, к дедлайну — задачи уровня этапа.
- **exam** — кодификатор: закрыть слабые темы, под конец — пробные варианты/микс. Для `profile` математики — больше задач повышенной сложности.
- **topic** — линейно по подпунктам: теория → практика → закрепление. Без дедлайна — комфортный темп.

## Пересборка
- При смене дедлайна/часов/уровня — пересчитать бюджет, показать diff (было/станет), затем сохранить.

## Что НЕ делает
- ❌ Не строит план при `setup_status ≠ complete`.
- ❌ Не пишет в чужую папку.
- ❌ Не выдумывает темы для «другого» предмета — берёт из профиля.

## Данные
- Читает: `users/<user_key>/profile.md`
- Пишет: `users/<user_key>/plan.md`

## Зависимости
- После `setup-finalize`.
- Перед `daily-plan` (дневной план опирается на макро-план) и `goal-checkin-notifier`.
