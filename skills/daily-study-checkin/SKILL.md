# daily-study-checkin

Ежедневный ритуал фиксации прогресса. Без чек-инов агент не знает, готовится ли пользователь → не работают трекинг и напоминания.

## Триггер
- Авто: 21:00 локального времени (или `checkin_time` из профиля)
- Вручную: «чек-ин», «итог дня», «как прошёл день», «отчёт»
- Требует `setup_status: complete`

## Поток
1. Спросить: **что изучил сегодня?** (темы, задачи, время)
2. Спросить: **что на завтра?** есть ли блокеры?
3. Записать в `users/<user_key>/progress.md` → раздел даты
4. Обновить **streak** (дней подряд с активностью) там же
5. Если streak сорвался — мягко отметить, без давления; предложить микро-цель на завтра
6. Если тема из `plan.md` закрыта — отметить её закрытой в `progress.md`

## Шаблон `users/<user_key>/progress.md`

```markdown
# Прогресс — <name>

**Streak:** <N> дней · **Обновлён:** YYYY-MM-DD

## Закрытые темы
- [x] <тема> — YYYY-MM-DD

## Дневник
### YYYY-MM-DD
- Изучил: ...
- Время: ... мин
- На завтра: ...
- Блокеры: ...
```

## Стиль
- Тёплый, без «молодец»-filler. По одному вопросу; если торопится — оба сразу.
- «Ничего не успел» → уточнить причину без осуждения, предложить реалистичный объём.

## Что НЕ делает
- ❌ Не выставляет баллы/оценки.
- ❌ Не сравнивает с другими.
- ❌ Не шлёт сами напоминания (это `goal-checkin-notifier`).
- ❌ Не пишет в чужую папку.

## Данные
- Пишет: `users/<user_key>/progress.md`
- Читает: `users/<user_key>/plan.md` (какие темы закрывать)

## Зависимости
- После `setup-finalize` + `study-plan`.
- Кормит данными `task-tracker` и `goal-checkin-notifier`.

## Где живёт реальное исполнение
**`SOUL.md` → секция «Рабочий режим»** — то, что агент реально выполняет. Этот скилл — дизайн-документ.
