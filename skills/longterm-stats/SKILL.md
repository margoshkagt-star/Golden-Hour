# longterm-stats

> Долговременная статистика по задачам конкретного пользователя. Требует `setup_status: complete`.
> Агрегаты за неделю / месяц / год / всё время, прогресс по долговременным задачам, дедлайны (только для долговременных).
> **Читает** атрибуты (вес, дедлайн, длительность, тип) из `users/<user_key>/tasks.yaml`.
> **Не обновляет** ничего в источнике — это чисто скилл-агрегатор и рендерер.
> **Рендерит** статистику в чат и в дашборд Obsidian.

---

## 🎯 Когда подключать

- Пользователь явно просит: «статистика за неделю», «итог месяца», «прогресс за год», «сколько всего сделано»
- Сработал авто-триггер: конец недели / конец месяца (cron)
- Сработал `day-end` скилла `task-tracker` — нужно обновить долговременный агрегат
- Пользователь закрывает долговременную задачу — обновить агрегаты

## ⏰ Авто-триггеры

| Триггер | Шаблон | Когда |
|---------|--------|-------|
| Воскресенье 22:00 | `weekly` + `render` | cron |
| Последний день месяца 22:00 | `monthly` + `render` | cron |
| 31 декабря 22:00 | `yearly` + `render` | cron |
| При `day-end` от `task-tracker` | лёгкое обновление `all-time` | hook |
| Закрытие долговременной задачи | `long-tasks` + `render` | event |

Каналы: cron → `systemEvent` в main session; ручной вызов → сообщение в чат.

## 📥 Источник данных

Файл: `users/<user_key>/tasks.yaml`

```yaml
tasks:
  - id: int
    name: string
    category: string
    weight: int              # 1-10
    deadline: ISO datetime
    status: planned | in_progress | done | blocked | overdue
    progress: 0-100
    actual_duration: int     # минуты, null если не отслеживалось
    estimated_duration: int  # минуты (опционально)
    task_type: short | long  # ОБЯЗАТЕЛЬНО для этого скилла
    closed_at: ISO datetime  # когда перешла в done

meta:
  ...
```

**⚠️ Требование к источнику:** у каждой задачи должен быть атрибут `task_type: short | long`.
Если `task_type` отсутствует — скилл использует fallback: считает долговременной любую задачу с `deadline > today + 7 дней`.

**Граница ответственности:** этот скилл **только читает** данные. Не создаёт задачи, не меняет атрибуты, не закрывает задачи. Запись — на `task-tracker` / скилле целеполагания.

## 🪟 Дашборд

Файл: `users/<user_key>/longterm-stats.md` (если у пользователя настроена синхронizaция с Obsidian — можно дублировать в vault).

**Это файл долгосрочной статистики.** Никаких отдельных спеков, заметок, связанных документов.

Авто-обновляется:
- раз в сутки (после `day-end` скилла `task-tracker`)
- по итогам недели / месяца / года
- при ручной команде «обнови долговременный дашборд»

Содержит:
- метку «Авто-обновляется Золотым часом» + дату/время обновления
- таблицу «Прогресс по периодам» (неделя / месяц / год / всё время) в двух метриках: по весу и в часах
- раздел «Долговременные задачи» — список + прогресс
- раздел «🔥 Дедлайны долговременных» — ТОЛЬКО долговременные
- пояснения по формулам

**Без YAML-фронтматтера, без таблицы метаданных, без служебных секций** — только полезный вывод. Метаданные (type/skill/source/renderer) хранятся в workspace MEMORY.md, а не в файле.

**Ручное обновление:** команда «обнови долговременный дашборд».

## 🧮 Формулы

### Прогресс по весу за период

```
period_progress_weight = Σ(weight × progress) / Σ(weight)  для задач периода
```

### Прогресс в часах за период

```
period_progress_hours = Σ(actual_duration) / Σ(estimated_duration)  для задач периода
```

Если `estimated_duration` отсутствует у большинства задач — показывать только метрику по весу, а в часах — факт: `Σ(actual_duration)` без знаменателя.

### Средний прогресс долговременных

```
long_avg = Σ(weight × progress) / Σ(weight)  для задач с task_type=long
```

### Границы периодов

- Неделя: последние 7 дней от текущего дня (включая сегодня)
- Месяц: последние 30 дней (или календарный месяц, если команда «месяц» пришла 1-го числа)
- Год: последние 365 дней (или календарный год)
- Всё время: все задачи с `created_at` или с начала трекера

### Под риском (для долговременных)

```
risk = (planned_progress - actual_progress) / planned_progress > 0.25
```

## ⚠️ Что скилл НЕ делает (явно)

- Не показывает краткосрочные задачи в дедлайнах (→ `task-tracker` → `urgent`)
- Не обновляет `users/<user_key>/tasks.yaml` (только читает)
- Не создаёт задачи, не меняет атрибуты (→ скилл целеполагания)
- Не валидирует корректность `task_type` (ответственность источника)
- Не отправляет напоминания (→ `goal-checkin-notifier`)

## 📋 Демо

Демо-датасет на 15 задач (3 категории, веса 1–6, статусы mixed) лежит в `users/<user_key>/tasks.yaml` — используется `task-tracker` для проверки шаблонов. Для `longterm-stats` отдельный демо-датасет не нужен: скилл берёт те же задачи и фильтрует по `task_type: long`. Если таких нет — в дашборде будут нули (см. текущее состояние).
