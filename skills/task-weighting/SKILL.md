# task-weighting

Единая система оценки **важности** и **сложности**. Цель — чтобы план ставил важное вперёд, а день не получался перегруженным тяжёлым.

## Шкала важности (приоритет) — 1…5
| Балл | Значение |
|---|---|
| 1 | факультатив, можно пропустить |
| 2 | низкая |
| 3 | обычная (по умолчанию) |
| 4 | высокая — нужно к цели |
| 5 | критично — решающая тема / горящий дедлайн |

Хранится как заданная пользователем база: `priorities: {<тема|предмет>: 1-5}`.

## Шкала сложности — 1…5
| Балл | Значение |
|---|---|
| 1 | очень легко |
| 2 | легко |
| 3 | средне (по умолчанию) |
| 4 | тяжело |
| 5 | очень тяжело |

База: `difficulty: {<тема>: 1-5}` (если пользователь не указал — считать 3).

## Эффективный приоритет (на момент планирования)
```
eff_priority = clamp( base_priority + deadline_boost + weakness_boost , 1, 5 )
```
- `deadline_boost`: +2 если до дедлайна < 3 недель; +1 если < 8 недель; иначе 0
- `weakness_boost`: +1 если уровень темы `zero`/`weak`/«с нуля»/«новичок»
- Идея: слабая + важная + близкий дедлайн → наверх.

## Эффективная сложность (на момент планирования)
```
eff_difficulty = clamp( base_difficulty + level_adjust , 1, 5 )
```
- `level_adjust`: +1 если уровень темы низкий (`zero`/`weak`); −1 если высокий (`strong`/`expert`/«продвинутый»/«топ»)
- Идея: тема, в которой пользователь слаб, ощущается тяжелее — учитываем при балансировке дня.

## Дневной бюджет сложности (нагрузка)
Из `daily_load` профиля (переносимая нагрузка):
| `daily_load` | Бюджет дня `D_max` (сумма сложностей блоков) |
|---|---|
| `light` (щадящий) | 6 |
| `normal` (обычный) | 9 |
| `intense` (интенсивный) | 12 |

Если `daily_load` не задан — `normal`. Бюджет можно слегка масштабировать под часы конкретного дня (меньше часов → меньше бюджет).

## Что отдаёт наружу
Для любого набора тем/задач — список с полями `{title, eff_priority, eff_difficulty, est_minutes}`. Это вход для `daily-balancer` (сборка дня) и `study-plan` (порядок тем).

## Данные
- Читает: `users/<user_key>/profile.md` → `priorities`, `difficulty`, `daily_load`, уровни тем, `deadline`
- Базовые баллы важности/сложности задаются в онбординге (`setup-finalize`) и правятся командами «<тема> важнее/менее важно», «<тема> легче/тяжелее».

## Зависимости
- Используется `daily-balancer` и `study-plan`.
- Базовые значения собирает `setup-finalize`.

## Где живёт реальное исполнение
**`SOUL.md` → секции «Финал настройки» и «Рабочий режим»** — то, что агент реально выполняет. Этот скилл — дизайн-документ.
