# task-tracker

> Трекер задач с отслеживанием: расчёт прогресса, контрольные точки, напоминания, итоги.
> **Читает** атрибуты (вес, дедлайн, категория) из `state/tasks.yaml`.
> **Обновляет** только `status`, `progress`, `actual_duration`, `updated_at`.
> **Рендерит** статистику в чат и в дашборд Obsidian.

---

## 🎯 Когда подключать

- Пользователь явно просит статистику: «прогресс», «что горит?», «напомни», «итог дня»
- Сработал авто-триггер: приближающийся дедлайн, КТ, конец дня, периодический отчёт
- Пользователь отчитался о прогрессе — нужно обновить state и дашборд

| Команда | Действие |
|---------|----------|
| «прогресс», «что по прогрессу?» | шаблон `progress` |
| «что горит?», «горит?» | шаблон `urgent` |
| «напомни про X» | шаблон `reminder` для X |

| «статистика», «отчёт» | полная сводка |

| «сделала X», «Y на 50%», «Z просрочена» | обновить state + дашборд |

## ⏰ Авто-триггеры

| Триггер | Шаблон | Когда |
|---------|--------|-------|
| За 5 мин до старта задачи | `reminder` | по расписанию |
| За 30 мин / 2 ч / 4 ч до дедлайна | `urgent`-лайт | по дедлайну |
| После дедлайна (если не сделано) | `urgent` + «просрочено» | повтор каждый час |
| В момент КТ | `checkpoint` | по расписанию |
| В 22:00 или при 2 ч бездействия | `day-end` | cron |
| Каждые 2 ч активной работы | короткий `progress` | cron |

Каналы: cron → `systemEvent` в main session; ручной вызов → сообщение в чат.

## 📥 Источник данных

Файл: `state/tasks.yaml`

```yaml
tasks:
  - id: int
    name: string
    category: string
    weight: int            # 1-10
    deadline: ISO datetime
    status: planned | in_progress | done | blocked | overdue
    progress: 0-100
    actual_duration: int   # минуты, null если не отслеживалось

meta:
  work_start: "HH:MM"
  work_end: "HH:MM"
  updated_at: ISO datetime
```

**Граница ответственности:** этот скилл **только читает** атрибуты (`name`, `category`, `weight`, `deadline`) и **обновляет** `status`, `progress`, `actual_duration`, `updated_at`. Создание задач, изменение веса/дедлайна/категории — на скилле целеполагания. Планирование последовательности — на скилле планирования.

| # | Задача | Дедлайн | Осталось    | Вес | Статус      |
|---|--------|---------|-------------|-----|-------------|
| N | ...    | HH:MM   | 🔥 −X мин   | W   | overdue     |

🔥 Просрочено: N

При попадании:

При не-попадании:
```
⏰ КТ HH:MM — «описание»

⚠️ Статус: под угрозой
📉 Отставание: ...
🔧 Рекомендация: ...
```

```

📈 Сделано: X/Y (Z%)
✅ Категория 1: X/Y (Z%) 🏆
🟡 Категория 2: X/Y (Z%)

⏰ Контрольные точки:
✅ HH:MM — попадание
❌ HH:MM — не успели

📝 Переносится на завтра:
— ...
```

### 6. Полная сводка по категориям

```
🎓 Категория 1 (N задач)  [█████░░░░░] X%
  ▶️  Задача A   60%  ⚠️ 25 мин
  ⏳  Задача B    0%

💼 Категория 2 (N задач)  [███████░░░] Y%
  ...
```

### 7. Легенда

| Символ | Статус | Цвет |
|--------|--------|------|
| ⏳ | planned | ⚪ |
| ▶️ | in_progress | 🔵 |
| ✅ | done | 🟢 |
| 🚫 | blocked | ⚫ |
| 🔥 | overdue | 🔴 |

🟢 в графике · 🟡 под риском (≤ 25% отставания) · 🔴 просрочено

Прогресс-бар: 20 символов, `[█]` заполнено, `[░]` нет.

## 🪟 Дашборд в Obsidian

Файл: `📦 OpenClaw\Дашборд — Трекер задач.md`

**Это единственный Obsidian-файл по трекеру.** Никаких отдельных спеков, заметок, связанных документов.

Авто-обновляется при каждом отчёте пользователя. Содержит:
- метку «Авто-обновляется Голди» + дату/время обновления
- счётчик рабочего времени: Прошло / Осталось / Время работы
- общий прогресс + прогресс-бар
- разбивка по категориям
- счётчик статусов
- блок «🔥 Горит»
- полный список всех задач с %

**Без YAML-фронтматтера, без таблицы метаданных, без служебных секций** — только полезный вывод. Метаданные трекера (type/skill/source/renderer) хранятся в workspace MEMORY.md, а не в файле.

**Ручное обновление:** команда «обнови дашборд».

## 🧮 Формулы

> ⚠️ **Ключевое правило: общий прогресс ВСЕГДА считается по весу, не по количеству задач.** Тяжёлая задача влияет на общий % сильнее, чем несколько лёгких.

### Общий прогресс (единственный метод — по весу)

```
overall = Σ(weight × progress) / Σ(weight)
```

**Пример расчёта** (3 задачи с разными весами):

| Задача | Вес | Прогресс | weight × progress |
|--------|-----|----------|-------------------|
| A      | 1   | 100%     | 100               |
| B      | 5   | 60%      | 300               |
| C      | 1   | 0%       | 0                 |
| **Σ**  | **7** |       | **400**           |

`overall = 400 / 7 ≈ 57%`

Для сравнения: **простое среднее по количеству** (НЕ используется) дало бы `(100+60+0)/3 ≈ 53%`. Разница заметна, когда веса сильно разные.

При `weight = 1` для всех задач формула вырождается в среднее арифметическое — это частный случай, не основной режим.

### Прогресс по категории (тоже по весу)

```
category_progress = Σ(weight × progress) / Σ(weight) в категории
```

### Под риском (риск-индикатор)

```
risk = (planned_progress - actual_progress) / planned_progress > 0.25
```

### Осталось до дедлайна

```
remaining = deadline - now  →  "X ч Y мин" / "🔥 −X мин" (если просрочено)
```

### Классификация по времени (для `urgent`)

- 🔥 overdue: время вышло, прогресс < 100%
- 🟡 под риском: осталось < 1.5 ч
- 🟢 в графике: осталось ≥ 1.5 ч

## ⚠️ Что скилл НЕ делает (явно)

- Не создаёт задачи (→ скилл целеполагания)
- Не меняет атрибуты: `weight`, `deadline`, `category`, `name` (→ скилл целеполагания)
- Не составляет план / последовательность (→ скилл планирования)
- Не валидирует корректность атрибутов (ответственность источника)
